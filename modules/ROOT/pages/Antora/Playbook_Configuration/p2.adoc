= Content Sources
:toc: left
:toclevels: 2
:sectnums:
:icons: font

Content sources define where Antora should retrieve documentation content from. These sources are usually Git repositories, but they can also be local directories.

Each source typically includes:

* Repository URL
* Branch or tag patterns
* Optional start paths within the repository

Example configuration:

[source,yaml]
----
content:
  sources:
    - url: https://gitlab.com/example/user-guide.git
      branches: main
    - url: https://gitlab.com/example/admin-docs.git
      branches: [v1.0, v2.0]
      start_path: docs
----

Explanation:

* `url` – Location of the content repository
* `branches` – Branches or tags containing documentation
* `start_path` – Directory within the repository where documentation begins

TIP: Antora can aggregate documentation from multiple repositories and combine them into a single documentation site.
